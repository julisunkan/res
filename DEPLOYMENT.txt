DEPLOYMENT INSTRUCTIONS — AI Resume & Cover Letter Creator
==========================================================
Target Platform: Render (render.com)
Database:        SQLite (stored on a Render persistent disk)


PREREQUISITES
-------------
- A Render account (Starter plan or higher — required for persistent disk)
- Your project code pushed to a GitHub or GitLab repository
- A Groq API key (free at https://console.groq.com)


PRICING NOTE
------------
This setup uses a Render persistent disk to keep the SQLite database
intact across deploys and restarts. Disks require the Starter plan:

  Web service (Starter):   $7 / month
  Persistent disk (1 GB):  $0.25 / month
  Total:              approx. $7.25 / month

No PostgreSQL add-on is needed.


STEP 1 — PUSH CODE TO GITHUB/GITLAB
-------------------------------------
Make sure all files are committed and pushed to your repository,
including render.yaml which automates the Render configuration.

Files that must be present in the repo:
  - render.yaml
  - requirements.txt
  - main.py
  - app.py
  - extensions.py
  - models/
  - routes/
  - utils/
  - templates/
  - static/

Files excluded from the repo (via .gitignore — do NOT push these):
  - *.db  (your local SQLite database — the server gets its own copy)
  - uploads/
  - instance/
  - .env


STEP 2 — CREATE A NEW BLUEPRINT ON RENDER
------------------------------------------
1. Log in to your Render dashboard at https://dashboard.render.com
2. Click "New" in the top right corner
3. Select "Blueprint"
4. Connect your GitHub/GitLab account if not already connected
5. Search for and select your repository
6. Click "Connect"

Render will detect render.yaml automatically and show you a preview
of what will be created:
  - A web service named "ai-resume-app" (Starter plan)
  - A 1 GB persistent disk mounted at /data for the SQLite database


STEP 3 — SET ENVIRONMENT VARIABLES
------------------------------------
During the Blueprint setup, Render will prompt you for any variables
marked as "sync: false". You will need to enter:

  GROQ_API_KEY    Your Groq API key from https://console.groq.com
                  This powers all AI features (resume generation,
                  cover letters, interview prep, career chat, etc.)
                  Without this key the app runs but AI features
                  will not work.

The following are handled automatically by Render:
  SECRET_KEY      Auto-generated by Render (no action needed)
  DATABASE_URL    Pre-set in render.yaml to sqlite:////data/resume_app.db


STEP 4 — DEPLOY
----------------
1. Click "Apply" to start the deployment
2. Render will:
   a. Provision the 1 GB persistent disk at /data
   b. Install Python dependencies from requirements.txt
   c. Start the app with gunicorn on the assigned port
3. On first startup the app creates all database tables automatically
4. Deployment takes approximately 3–5 minutes on first run
5. When the status shows "Live", click the service URL to verify


STEP 5 — MIGRATE YOUR LOCAL DATA (OPTIONAL)
---------------------------------------------
If you want to carry over data from your local SQLite database:

1. In the Render dashboard, go to your web service
2. Click "Shell" in the top navigation
3. Run the following to confirm the disk is mounted:
     ls /data
4. Use the Render shell or the admin panel's database import feature
   to upload your local resume_app.db contents

Alternatively, start fresh — all tables are auto-created on first boot.


STEP 6 — POST-DEPLOYMENT CONFIGURATION
----------------------------------------
Once the app is live, complete these steps inside the app itself:

1. Visit https://your-app.onrender.com/julisunkan to access admin
2. Go to "Site Settings" and set:
     Site URL:   https://your-app.onrender.com
                 (this ensures sitemap.xml uses the correct domain)
3. Optionally configure:
     - App name and tagline
     - Contact email
     - Social media links
     - Google Analytics ID
     - Google Search Console verification tag
     - AdSense settings


DATABASE NOTES
--------------
- The SQLite database file lives at /data/resume_app.db on the
  persistent disk. It survives deploys, restarts, and crashes.
- All database tables are created automatically on first startup.
  No manual migration steps are required.
- Do not delete or unmount the disk — doing so will permanently
  erase all data. Back up the database via the admin export feature
  before making infrastructure changes.


UPDATING THE APP
----------------
After making code changes:
1. Commit and push to your GitHub/GitLab repository
2. Render automatically detects the push and redeploys
3. The /data disk is preserved across redeploys — no data is lost


ENVIRONMENT VARIABLES REFERENCE
---------------------------------
Variable        Required    Description
-----------     --------    -----------
SECRET_KEY      Yes         Flask session secret — auto-generated by Render
DATABASE_URL    Yes         SQLite path — pre-set in render.yaml
GROQ_API_KEY    Yes         Groq AI API key — must be set manually


TROUBLESHOOTING
---------------
App fails to start:
  - Check the Render logs tab for the error message
  - Ensure all files in the "Files that must be present" list above
    are committed to the repository

Database errors on first boot:
  - Tables are created automatically; wait for the first deploy to
    complete fully before testing
  - Confirm the disk is mounted by checking /data in the Render shell

AI features not working:
  - Confirm GROQ_API_KEY is set correctly in Render environment variables
  - Test your key at https://console.groq.com before entering it

Static files not loading:
  - Flask serves static files directly — no CDN setup needed
  - If assets appear broken, do a hard refresh (Ctrl+Shift+R)

Data missing after redeploy:
  - Verify the /data disk is still attached in the Render dashboard
    under your service -> Disks
  - Data loss only occurs if the disk is manually deleted or detached
