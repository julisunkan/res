📋 Setup Instructions

Follow these steps to connect your app to Google Cloud Firestore. This takes about 5 minutes.

1
Create a Firebase Project
Go to console.firebase.google.com and click Add project.
Enter a project name (e.g. resume-app-db), then follow the wizard. You can disable Google Analytics if you don't need it.
2
Enable Cloud Firestore
In the left sidebar of Firebase Console, click Build → Firestore Database.
Click Create database, choose Start in production mode, and select your preferred region (e.g. us-east1). Click Enable.
3
Generate a Service Account Key
Go to Project Settings (gear icon ⚙️ next to Project Overview).
Click the Service accounts tab.
Click Generate new private key → Generate key.
A .json file will download — this is your service account key.
4
Paste the Key Below & Save
Open the downloaded JSON file in any text editor, select all (Ctrl+A), copy (Ctrl+C), and paste it into the textarea in the Credentials section below. Click Save Credentials.
5
Update Firestore Security Rules
In Firebase Console → Firestore Database → Rules, replace the content with the following (allows server-side SDK only — no public access):
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
Click Publish. Your server uses the Admin SDK which bypasses these rules — this blocks any client-side access.
✓
Test Connection & Sync Data
Click Test Connection below. Once successful, use the Export to Firestore section to push your current database data into Firestore collections.
🔑 Service Account Credentials

Paste the full contents of your Firebase service account JSON file here. It will be stored securely in your database and never exposed in exports.

📤 Export Local Data → Firestore

Push data from your local database into Firestore collections. Existing Firestore documents with the same ID will be overwritten. Sensitive fields (API keys, passwords) are always excluded.

📥 Import Firestore Data → Local DB

Pull records from Firestore into your local database. Only records whose ID doesn't already exist locally will be added (no overwrites). Useful for restoring from Firestore or migrating between environments.

⚠ Import only adds missing records — it will not update or overwrite existing local data.
